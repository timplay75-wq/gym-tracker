# Промты для создания Gym Tracker PWA

## 📋 Этап 1: Исследование и планирование

### Промт 1.1: Анализ референсного приложения
```
Я хочу создать веб-приложение для трекинга тренировок, аналогичное приложению "Gym Tracker" из AppStore/Google Play. 

Помоги мне проанализировать ключевой функционал такого приложения:
1. Какие основные экраны и разделы должны быть?
2. Как организована структура тренировок (программы -> тренировки -> упражнения -> подходы)?
3. Какие метрики отслеживаются для упражнений?
4. Как выглядит UI/UX типичного фитнес-трекера?
5. Какие визуальные элементы и паттерны используются в современных фитнес-приложениях?

Предложи детальную структуру навигации и основных функций.
```

### Промт 1.2: Создание структуры данных
```
На основе анализа фитнес-приложения "Gym Tracker", создай TypeScript типы и интерфейсы для:

1. **WorkoutProgram** - программа тренировок (сплит)
   - Название программы
   - Список тренировок
   - Расписание (какие дни недели)
   - Дата создания

2. **Workout** - отдельная тренировка
   - Название (например: "Грудь и трицепс")
   - Список упражнений
   - Дата выполнения
   - Длительность
   - Статус (запланирована/выполнена)

3. **Exercise** - упражнение
   - Название
   - Категория мышечной группы
   - Тип (силовое/кардио/растяжка)
   - История подходов
   - Персональные рекорды (PR)

4. **Set** - подход
   - Вес (kg)
   - Повторения
   - Отдых (секунды)
   - RPE (оценка усилия)
   - Выполнен или нет
   - Заметки

5. **ExerciseStats** - статистика по упражнению
   - Общий тоннаж
   - Максимальный вес
   - Объем за период
   - График прогресса

Добавь связи между типами и экспортируй в файл types/index.ts
```

```
Настрой Tailwind CSS для приложения трекинга тренировок с поддержкой светлой и темной темы.
```
